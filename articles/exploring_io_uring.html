Since kernel version 5.1, Linux has a proper asynchronous io interface called io_uring whose design received a lot of praise in the Linux community. In a current project we aim for very high io throughput, which is why I want to have a closer look at it.

These notes are only about the low level io_uring api. For a higher level interface, it is recommended to use the liburing helper library which can be found here: https://github.com/axboe/liburing.

An io_uring instance provides two ring buffers. One for submitting operations and another one for completed operations. Both ring buffers are shared between kernel and user space need to be mapped to user space memory before we can use them. After having mapped the ring buffers, we can start to submit operations by putting them on the submission ring and wait for their completion by waiting for new elements being pushed to the completion ring by the kernel. Sounds fairly simple!

Let us begin, by initializing a new io_uring instance.

1struct io_uring_params params = {0};
2int ring_fd = io_uring_setup(1024, &params);

Here, we tell io_uring to create ring buffers that can hold up to 1024 operations. params will be filled with all the information that are needed to access variables in the ring buffer, which we're going to setup in the next step. The returned file descriptor ring_fd is used to refer to the io_uring instance.

io_uring provides three constants that tell us the address offsets for mapping the io_uring objects via mmap.

1IORING_OFF_SQ_RING  // addr offset of the submission ring
2IORING_OFF_CQ_RING  // addr offset of the completion ring
3IORING_OFF_SQES     // addr offset of the array of submission entries

While the completion ring contains the completion entry array, the submission ring only contains an array of indices to the submission entry array. The submission entry array itself is not part of the ring and must be mapped separately. The reason for having an indiraction array instead of indexing the submission entries directly is that it gives some more flexibility in using the submission ring.

Now, let's map the submission ring and entry array. The io_uring_param struct's sq_off field tells us how to access the ring's parameters.

 1struct sq_ring
 2{
 3    struct io_uring_sqe *sqes;
 4    unsigned int *head;
 5    unsigned int *tail;
 6    unsigned int *ring_mask;
 7    unsigned int *ring_entries;
 8    unsigned int *flags;
 9    unsigned int *dropped;
10    unsigned int *array;
11};
12
13static void sq_ring_init(struct sq_ring* sring, int ring_fd, struct io_uring_params *p)
14{
15    void *ptr;
16    ptr = mmap(NULL, p->sq_off.array + p->sq_entries * sizeof(unsigned int),
17               PROT_READ | PROT_WRITE, MAP_SHARED | MAP_POPULATE,
18               ring_fd, IORING_OFF_SQ_RING);
19    sring->head = ptr + p->sq_off.head;
20    sring->tail = ptr + p->sq_off.tail;
21    sring->ring_mask = ptr + p->sq_off.ring_mask;
22    sring->ring_entries = ptr + p->sq_off.ring_entries;
23    sring->flags = ptr + p->sq_off.flags;
24    sring->dropped = ptr + p->sq_off.dropped;
25    sring->array = ptr + p->sq_off.array;
26    sring->sqes = mmap(0, p->sq_entries * sizeof(struct io_uring_sqe),
27            PROT_READ | PROT_WRITE, MAP_SHARED | MAP_POPULATE,
28            ring_fd, IORING_OFF_SQES);
29}

Similarly, we are going to map the completion ring.

 1struct cq_ring
 2{
 3    unsigned int *head;
 4    unsigned int *tail;
 5    unsigned int *ring_mask;
 6    unsigned int *ring_entries;
 7    unsigned int *flags;
 8    struct io_uring_cqe *cqes;
 9};
10
11static void cq_ring_init(struct cq_ring* cring, int ring_fd, struct io_uring_params *p)
12{
13    void *ptr;
14    ptr = mmap(NULL, p->cq_off.cqes + p->cq_entries * sizeof(struct io_uring_cqe),
15               PROT_READ | PROT_WRITE, MAP_SHARED | MAP_POPULATE,
16               ring_fd, IORING_OFF_CQ_RING);
17    cring->head = ptr + p->cq_off.head;
18    cring->tail = ptr + p->cq_off.tail;
19    cring->ring_mask = ptr + p->cq_off.ring_mask;
20    cring->ring_entries = ptr + p->cq_off.ring_entries;
21    cring->flags = ptr + p->cq_off.flags;
22    cring->cqes = ptr + p->cq_off.cqes;
23}

That's it! Now we are ready to submit an operation. The most simple blocking operation I can think of is reading from standard input. So this is what are we going to do.

Let's write a function for submitting a read operation on stdin:

 1static void submit_read(struct sq_ring *sring, char *buf, size_t buf_len)
 2{
 3    unsigned int tail, index;
 4    tail = *sring->tail;
 5    index = tail & (*sring->ring_mask);
 6
 7    // Retrieve an sq entry
 8    struct io_uring_sqe *sqe = &sring->sqes[index];
 9
10    // Fill the sq
11    sqe->opcode = IORING_OP_READ;
12    sqe->fd = 0; // stdin fd
13    sqe->addr = (unsigned long long)buf;
14    sqe->len = buf_len;
15    sqe->flags = 0;
16    sqe->off = 0;
17    sqe->user_data = 0;
18
19    // Tell the indirection array where to find the sqe
20    sring->array[index] = index;
21
22    // Update the ring's tail
23    ++tail;
24    __atomic_store(sring->tail, tail, __ATOMIC_RELEASE)
25}

Note: The atomic_store call here ensures that the tail is written atomically and that the writes to the sqe are visible before the tail is updated. The kernel will read the tail value and then read the sqe. If the writes to the sqe are not visible before the tail is updated, the kernel may read the sqe before it is written. For more information on atomic operations, see this.

After submitting, we need to tell io_uring that there are new entries on the ring for it to consume. Then, we gotta wait for new entries on the completion ring. Both can be done with a single system call: Its signature looks like this

1int io_uring_enter(unsigned int fd, unsigned int to_submit,
2                   unsigned int min_complete, unsigned int flags,
3                   sigset_t *sig);

In our example we call it the following way:

1    io_uring_enter(ring_fd, 1, 1, IORING_ENTER_GETEVENTS, NULL)

Here, we are telling io_uring to submit one entry, and wait for one entry to be completed. If we didn't pass IORING_ENTER_GETEVENTS, then the system call wouldn't block until one operation completes.

We still need to pull the completed operation from the ring and read its result.

 1static int pull(struct cq_ring *cring)
 2{
 3    unsigned int head, tail, index;
 4    __atomic_load(cring->tail, &tail, __ATOMIC_AQUIRE);
 5    head = *cring->head;
 6    if (tail == head)
 7        return -1;
 8    index = head & (*cring->ring_mask);
 9
10    // Get the cqe from the ring
11    struct io_uring_cqe *cqe = &cring->cqes[index];
12
13    // cqe->res contains the result of our read operation
14    int r = cqe->res;
15
16    // Update the ring's head
17    ++head;
18    __atomic_store(cring->head, head, __ATOMIC_RELEASE)
19    return cqe->res;
20}

Similar to the sq ring, we need to make sure that the reads from the cqe are visible before the head is updated.

Now, let's put everything together:

 1#include <linux/io_uring.h>
 2#include <sys/mman.h>
 3#include <stdlib.h>
 4
 5/**
 6    Put here all our stuff from above
 7**/
 8
 9int main(int argc, char **argv)
10{
11    struct io_uring_params params = {0};
12    int ring_fd = io_uring_setup(1024, &params);
13    
14    struct sq_ring sring;
15    sq_ring_init(&sring, ring_fd, &params);
16    struct cq_ring cring;
17    cq_ring_init(&cring, ring_fd, &params);
18
19    char buffer[512];
20    submit_read(&sring, buffer, sizeof(buffer));
21    io_uring_enter(ring_fd, 1, 1, IORING_ENTER_GETEVENTS, NULL);
22    int r = pull(&cring);
23    if (r > 0)
24        printf("You Typed: %.*s", r, buffer);
25    return 0;
26}

Our program will simply wait for some input on the command line, print it and exit.

1→ ./io_uring_test
2Hello io_uring 
3You Typed: Hello io_uring

Additional Notes:

Depending on your C library, the io_uring system-calls io_uring_setup and io_uring_enter might not be available yet. We can define them ourselves, for example via the following snippet that I copied from the Linux Kernel.

 1#include <signal.h>
 2#include <sys/syscall.h>
 3
 4int io_uring_setup(unsigned int entries, struct io_uring_params *p)
 5{
 6	return syscall(__NR_io_uring_setup, entries, p);
 7}
 8
 9int io_uring_enter(int fd, unsigned int to_submit, unsigned int min_complete,
10		   unsigned int flags, sigset_t *sig)
11{
12	return syscall(__NR_io_uring_enter, fd, to_submit, min_complete,
13			flags, sig, _NSIG / 8);
14}
